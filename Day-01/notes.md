The Core Systems -  

Data: Raw, unorganized facts, figures, text, or symbols that lack context on their own (e.g., the number 43 or the word Roger). 

Database: An organized, electronic collection of data stored in a computer system so it can be easily accessed, managed, and updated. 

Database Management System (DBMS): The system software used to create, protect, read, update, and manage databases, serving as the interface between the data and the user. Examples include MySQL, PostgreSQL, and Oracle. 

The Storage Architecture- 

Schema: The logical blueprint or structural design that defines how the entire database is organized, including its tables, columns, and data relationships. 

Table: A structured collection of related data entries organized visually into vertical columns and horizontal rows (e.g., a Customers table). 

Column (Field / Attribute): A vertical section of a table that defines a specific type of information or characteristic that every entry must have (e.g., Email_Address or Age). 
Row (Tuple): A single horizontal alignment of data within a table representing an individual entry. 

Record: The complete set of data values gathered across all columns for a single specific row entity (e.g., all the personal profile data belonging to a single client). Note: In a relational database, "Row" and "Record" are frequently used interchangeably.
